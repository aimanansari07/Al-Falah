# Al-Falah Masjid App — Frontend

React 18 + Vite PWA, with Capacitor for native Android APK.

---

## PREREQUISITES

| Tool | Version |
|------|---------|
| Node.js | 18+ |
| npm | 9+ |
| Android Studio | Latest |
| Java JDK | 17+ |
| Android SDK | API 34 (Android 14) |

> **Min target:** API 22 (Android 5.1) — covers ~99% of devices.

---

## WEB DEVELOPMENT

```bash
cd frontend
npm install
npm run dev       # http://localhost:5173
npm run build     # production build → dist/
npm run preview   # preview production build
```

---

## ANDROID APK — FIRST TIME SETUP

```bash
# 1. Install dependencies
npm install

# 2. Build the React app
npm run build

# 3. Add the Android platform (run once)
npx cap add android

# 4. Sync web build into Android project
npx cap sync android

# 5. Open in Android Studio
npx cap open android
```

In Android Studio:
- Wait for Gradle sync to complete
- **Build → Generate Signed Bundle / APK → APK**
- Choose your keystore (or create one)
- Select **debug** or **release**

---

## BUILD APK — QUICK COMMANDS

```bash
# Debug APK (no signing required)
npm run apk:debug
# Output: android/app/build/outputs/apk/debug/app-debug.apk

# Release APK (requires keystore)
npm run apk:release
# Output: android/app/build/outputs/apk/release/app-release.apk

# Or run scripts directly
bash scripts/build-apk.sh
bash scripts/build-release-apk.sh
```

---

## CAPACITOR SYNC (after any web change)

```bash
npm run build
npm run cap:sync    # = npx cap sync android
```

---

## INSTALL VIA USB (recommended)

Connect your phone via USB with USB Debugging enabled, then:
```bash
npm run apk:debug   # builds + installs automatically via adb
```

ADB path: `~/Library/Android/sdk/platform-tools/adb`

To check your device is detected:
```bash
~/Library/Android/sdk/platform-tools/adb devices
```

## SHARE THE APK

1. Build: `npm run apk:debug`
2. Locate: `android/app/build/outputs/apk/debug/app-debug.apk`
3. Share via WhatsApp, Google Drive, or any file transfer
4. Recipients tap → **Install** (must enable "Install from unknown sources" once)

## PUSH NOTIFICATIONS

Push notifications use **FCM via `@capacitor/push-notifications`**.

- Permission is requested automatically on first launch
- FCM token is logged to console — wire it to your backend API when ready
- Notification tap navigates to the route specified in `notification.data.route`

To test notifications, use the Firebase Console:
- Project → Cloud Messaging → Send test message
- Set `data.route` to `/announcements`, `/prayers`, etc.

## TECH STACK

- **React 18** + **Vite 5**
- **Tailwind CSS 3**
- **Framer Motion** — page transitions + animations
- **React Router v6**
- **dayjs** + **hijri-calendar-system** — Hijri date conversion
- **vite-plugin-pwa** — PWA manifest + service worker
- **@capacitor/core** + **@capacitor/android** — Native Android wrapper
- **@capacitor/push-notifications** — FCM push notifications
- **@capacitor/local-notifications** — Local prayer reminders
- **@capacitor/share** — Native share sheet
- **@capacitor/app** — Back button handling + app lifecycle
